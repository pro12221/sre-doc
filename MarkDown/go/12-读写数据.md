# Go 读写数据

程序的价值在于处理数据，而数据从哪里来、到哪里去？这一章就讲 Go 怎么跟外界打交道——从键盘读输入、从文件读写数据、解析 JSON/XML、处理命令行参数等。掌握这些，你的程序就能"开口说话、竖起耳朵"了。

---

## 1. 读取键盘输入

### 1.1 最简单的方式：fmt.Scan 系列

就像填表格一样，`fmt` 包提供了几个函数让你从键盘一行一行地读数据：

| 函数 | 干什么 |
|------|--------|
| `fmt.Scanln(&a, &b)` | 读空格分隔的值，遇到换行停止 |
| `fmt.Scanf("%d %s", &a, &b)` | 按格式读，和 Printf 是一对 |
| `fmt.Sscanf(str, "%d", &a)` | 从字符串里按格式读，不是从键盘 |

```go
package main

import "fmt"

func main() {
    var firstName, lastName string
    fmt.Println("请输入你的姓名（空格分隔）：")
    fmt.Scanln(&firstName, &lastName)
    fmt.Printf("你好，%s %s！\n", firstName, lastName)
}
```

输入 `张 三`，输出 `你好，张 三！`

**注意**：`Scanln` 遇到空格就会把值分给下一个变量，所以它适合读"单词"，不适合读整句话。

### 1.2 更灵活的方式：bufio.Reader

如果你想读一整行（包括空格），用 `bufio.Reader` 更靠谱：

```go
package main

import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    // 创建一个"读取器"，绑定到键盘输入
    reader := bufio.NewReader(os.Stdin)
    fmt.Println("请随便输入点什么：")

    // 一直读到用户按回车（\n）为止
    input, err := reader.ReadString('\n')
    if err != nil {
        fmt.Println("读取出错：", err)
        return
    }
    fmt.Printf("你输入了：%s", input)
}
```

`ReadString('\n')` 的意思是：一直读，直到碰到换行符 `\n`，把读到的所有内容（含换行符）都返回给你。

> **要点速记**：
> - 简单场景用 `fmt.Scanln`，读整行用 `bufio.NewReader(os.Stdin).ReadString('\n')`
> - `os.Stdin` 就是键盘，`os.Stdout` 就是屏幕，`os.Stderr` 是错误输出（通常也是屏幕）

## 2. 文件读取

### 2.1 基本流程：打开 → 读 → 关闭

读文件就像去图书馆借书：先找到书（打开），再看书（读取），最后还回去（关闭）。

```go
package main

import (
    "bufio"
    "fmt"
    "io"
    "os"
)

func main() {
    // 第一步：打开文件
    file, err := os.Open("data.txt")
    if err != nil {
        fmt.Println("打开文件失败：", err)
        return
    }
    // 第三步（提前写）：确保最后关闭文件
    defer file.Close()

    // 第二步：逐行读取
    reader := bufio.NewReader(file)
    for {
        line, err := reader.ReadString('\n')
        fmt.Print(line) // 打印读到的一行
        if err == io.EOF {
            break // 文件读完了
        }
        if err != nil {
            fmt.Println("读取出错：", err)
            break
        }
    }
}
```

关键点：
- `os.Open()` 只读模式打开，返回 `*os.File`
- `defer file.Close()` 确保函数退出时关闭文件，**永远别忘了写这行**
- `io.EOF` 表示"文件到头了"，不是错误，是正常结束信号

### 2.2 一次性读完整个文件

如果文件不大，用 `os.ReadFile`（Go 1.16+）最省事：

```go
data, err := os.ReadFile("data.txt")
if err != nil {
    fmt.Println("读取失败：", err)
    return
}
fmt.Println(string(data)) // data 是 []byte，转成 string 打印
```

一行搞定，不用手动打开/关闭。就像"快递送到家"，不用自己去取。

### 2.3 按列读取数据

如果文件是表格形式的（空格或固定列数），可以用 `fmt.Fscanln`：

```go
// 文件内容：
// Apple  10  3.5
// Banana 5   2.8

file, _ := os.Open("products.txt")
defer file.Close()

for {
    var name string
    var count int
    var price float64
    _, err := fmt.Fscanln(file, &name, &count, &price)
    if err != nil {
        break
    }
    fmt.Printf("商品：%s 数量：%d 单价：%.2f\n", name, count, price)
}
```

> **要点速记**：
> - 小文件用 `os.ReadFile()`，大文件用 `bufio.Reader` 逐行读
> - 打开文件后第一件事就是写 `defer file.Close()`
> - `io.EOF` 是文件读完的正常信号，不是错误

## 3. 文件写入

### 3.1 基本流程：创建/打开 → 写 → 关闭

写文件就像往笔记本上写字：先翻到那一页（打开），再写内容，最后合上（关闭）。

```go
package main

import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    // 以"只写+创建"模式打开文件，0666 是权限（Unix 下生效）
    file, err := os.OpenFile("output.txt", os.O_WRONLY|os.O_CREATE, 0666)
    if err != nil {
        fmt.Println("创建文件失败：", err)
        return
    }
    defer file.Close()

    // 创建带缓冲的写入器
    writer := bufio.NewWriter(file)
    for i := 0; i < 5; i++ {
        writer.WriteString(fmt.Sprintf("第 %d 行\n", i+1))
    }
    writer.Flush() // 别忘了刷缓冲！否则内容可能没写进去
}
```

`os.OpenFile` 的常用标志：

| 标志 | 含义 |
|------|------|
| `os.O_RDONLY` | 只读 |
| `os.O_WRONLY` | 只写 |
| `os.O_CREATE` | 文件不存在就创建 |
| `os.O_TRUNC` | 文件存在就清空 |
| `os.O_APPEND` | 追加到文件末尾 |

### 3.2 简单写入

如果只是写点简单内容，不需要缓冲区：

```go
// 直接写屏幕
os.Stdout.WriteString("hello world\n")

// 直接写文件
f, _ := os.OpenFile("test.txt", os.O_CREATE|os.O_WRONLY, 0644)
defer f.Close()
f.WriteString("写入文件的内容\n")
```

或者用 `fmt.Fprintf`，它可以把内容写到任何实现了 `io.Writer` 的地方：

```go
fmt.Fprintf(f, "名字：%s 年龄：%d\n", "小明", 18)
```

> **要点速记**：
> - `os.OpenFile` 打开文件时可以组合多个标志用 `|` 连接
> - 用 `bufio.Writer` 写完后必须调 `Flush()`，否则数据可能留在缓冲区里没写入
> - `fmt.Fprintf(文件, 格式, 值...)` 可以直接格式化写入文件

## 4. 文件拷贝

拷贝文件就像接水管：一头接源文件，一头接目标文件，数据从一头流到另一头。

```go
func CopyFile(dstName, srcName string) (written int64, err error) {
    src, err := os.Open(srcName)
    if err != nil {
        return
    }
    defer src.Close()

    dst, err := os.OpenFile(dstName, os.O_WRONLY|os.O_CREATE, 0644)
    if err != nil {
        return
    }
    defer dst.Close()

    return io.Copy(dst, src) // 核心就这一行！
}
```

`io.Copy(dst, src)` 自动把 src 的全部内容写到 dst，返回拷贝的字节数。

**注意**：即使打开目标文件时出错了，`defer src.Close()` 仍然会执行，不会造成源文件泄漏。

> **要点速记**：
> - 文件拷贝用 `io.Copy(dst, src)`，不要自己手写循环
> - 打开文件后立即 `defer Close()`，即使后续出错也能正确关闭

## 5. 命令行参数

### 5.1 简单方式：os.Args

`os.Args` 是一个字符串切片，存着命令行参数。`os.Args[0]` 是程序名本身，`os.Args[1:]` 才是用户传的参数。

```go
package main

import (
    "fmt"
    "os"
    "strings"
)

func main() {
    if len(os.Args) > 1 {
        fmt.Println("你好，", strings.Join(os.Args[1:], " "))
    } else {
        fmt.Println("你好，陌生人")
    }
}
```

运行 `./hello 张三 李四`，输出 `你好，张三 李四`

### 5.2 正经方式：flag 包

`flag` 包可以定义带名字、默认值和帮助信息的命令行选项，就像专业的命令行工具那样：

```go
package main

import (
    "flag"
    "fmt"
)

func main() {
    // 定义一个 -n 标志，默认 false，帮助信息"换行输出"
    newLine := flag.Bool("n", false, "换行输出")
    // 定义一个 -count 标志，默认 1
    count := flag.Int("count", 1, "重复次数")

    flag.Parse() // 必须调用！解析命令行参数

    for i := 0; i < *count; i++ {
        if *newLine {
            fmt.Println("Hello")
        } else {
            fmt.Print("Hello ")
        }
    }
}
```

运行示例：
- `./prog` → 输出 `Hello `
- `./prog -n` → 输出 `Hello`（带换行）
- `./prog -count 3` → 输出 `Hello Hello Hello `

> **要点速记**：
> - 简单需求用 `os.Args`，专业需求用 `flag` 包
> - `flag.Parse()` 必须调用，之后才能使用 flag 的值
> - flag 值是指针类型（`*bool`、`*int`、`*string`），用的时候要加 `*`

## 6. JSON 数据格式

JSON 是互联网上最流行的数据交换格式，Go 对它的支持非常成熟。

### 6.1 结构体转 JSON（序列化）

```go
type Student struct {
    Name  string `json:"name"`   // json 标签指定 JSON 中的字段名
    Age   int    `json:"age"`
    Email string `json:"email,omitempty"` // omitempty：空值就不输出
}

stu := Student{Name: "小明", Age: 18}
data, err := json.Marshal(stu)
if err != nil {
    fmt.Println("序列化失败：", err)
    return
}
fmt.Println(string(data))
// 输出：{"name":"小明","age":18}  （Email 为空所以没输出）
```

如果要格式化输出（带缩进），用 `json.MarshalIndent`：

```go
data, _ := json.MarshalIndent(stu, "", "  ")
fmt.Println(string(data))
// 输出：
// {
//   "name": "小明",
//   "age": 18
// }
```

### 6.2 JSON 转结构体（反序列化）

```go
jsonStr := `{"name":"小红","age":20,"email":"hong@test.com"}`

var stu Student
err := json.Unmarshal([]byte(jsonStr), &stu)
if err != nil {
    fmt.Println("反序列化失败：", err)
    return
}
fmt.Printf("%+v\n", stu)
// 输出：{Name:小红 Age:20 Email:hong@test.com}
```

### 6.3 动态 JSON

如果 JSON 结构不确定，可以用 `map[string]interface{}`：

```go
jsonStr := `{"name":"小明","scores":{"math":95,"english":87}}`

var result map[string]interface{}
json.Unmarshal([]byte(jsonStr), &result)
fmt.Println(result["name"])           // 小明
fmt.Println(result["scores"].(map[string]interface{})["math"]) // 95
```

> **要点速记**：
> - 结构体字段加 `json:"字段名"` 标签，控制 JSON 输出的 key 名
> - `json.Marshal` 序列化，`json.Unmarshal` 反序列化
> - `omitempty` 标签让空值字段不输出
> - 不确定结构时用 `map[string]interface{}` 接收

## 7. XML 数据格式

XML 比 JSON 更"啰嗦"，但在配置文件和一些老系统中还在用。用法和 JSON 类似：

```go
type Person struct {
    XMLName xml.Name `xml:"person"`
    Name    string   `xml:"name"`
    Age     int      `xml:"age"`
}

// 结构体 → XML
p := Person{Name: "小明", Age: 18}
data, _ := xml.Marshal(p)
fmt.Println(string(data))
// 输出：<person><name>小明</name><age>18</age></person>

// XML → 结构体
var p2 Person
xml.Unmarshal(data, &p2)
fmt.Printf("%+v\n", p2)
```

`xml.MarshalIndent` 同样可以格式化输出。

> **要点速记**：
> - XML 用 `encoding/xml` 包，用法和 JSON 几乎一样
> - 结构体必须有个 `XMLName xml.Name` 字段指定根元素名
> - 标签格式是 `xml:"元素名"`

## 8. Gob 编码

Gob 是 Go 特有的二进制编码格式，比 JSON 更快更小，但只有 Go 程序能读。就像"方言"——只有自己人听得懂，但交流效率特别高。

```go
type Data struct {
    Name string
    List []int
}

// 编码
var buf bytes.Buffer
enc := gob.NewEncoder(&buf)
d := Data{Name: "测试", List: []int{1, 2, 3}}
enc.Encode(d)
fmt.Printf("Gob 编码后 %d 字节\n", buf.Len())

// 解码
var result Data
dec := gob.NewDecoder(&buf)
dec.Decode(&result)
fmt.Printf("%+v\n", result)
```

> **要点速记**：
> - Gob 是 Go 专用的二进制格式，跨语言场景不要用
> - 适合 Go 程序之间的数据传输（如 RPC）
> - 编码用 `gob.NewEncoder`，解码用 `gob.NewDecoder`

## 9. 密码学基础

Go 的 `crypto` 标准库提供了常用加密功能，不用自己造轮子：

| 包 | 用途 |
|----|------|
| `crypto/md5` | MD5 哈希（已不推荐用于安全场景，但校验文件还行） |
| `crypto/sha1` | SHA-1 哈希 |
| `crypto/sha256` | SHA-256 哈希（推荐） |
| `crypto/aes` | AES 对称加密 |
| `crypto/rsa` | RSA 非对称加密 |

计算 SHA-256 哈希的例子：

```go
import (
    "crypto/sha256"
    "fmt"
)

func main() {
    data := []byte("Hello, World!")
    hash := sha256.Sum256(data)
    fmt.Printf("%x\n", hash) // 输出十六进制哈希值
}
```

> **要点速记**：
> - 新项目用 SHA-256，别用 MD5/SHA-1 做安全校验
> - 哈希是单向的（不可逆），加密是双向的（可解密）
> - `crypto` 是标准库，不需要第三方包

## 新手常见坑

1. **写文件后没调 Flush()**：`bufio.Writer` 写的内容还在内存缓冲区里，程序就结束了，数据丢失。解决办法：写完必调 `writer.Flush()`。

2. **打开文件没 defer Close()**：文件句柄会泄漏，最终耗尽系统资源。打开后第一行就写 `defer file.Close()`。

3. **os.Args 下标越界**：直接用 `os.Args[1]` 但没判断长度，用户不传参数就 panic。先 `if len(os.Args) > 1` 检查。

4. **JSON 标签写错**：`json:"name"` 写成 `json:name` 或者忘了引号，导致字段名对不上。注意是反引号里用双引号。

5. **io.EOF 当错误处理**：`io.EOF` 表示文件读完，是正常的，不是错误。判断时要用 `if err == io.EOF` 单独处理。

6. **Windows 换行符问题**：Windows 换行是 `\r\n`，Unix 是 `\n`。`ReadString('\n')` 在两个系统上都能工作，但 `switch input { case "hello\n": }` 可能匹配不上 Windows 的输入。用 `strings.TrimSpace()` 去掉首尾空白更安全。
